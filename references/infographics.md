# Infographic and chart framework

## Choose one organizing category

Classify the infographic before drawing it:

1. **Compare two things** — show difference, overlap, trade-offs, before/after, or opposing positions.
2. **Story order** — show chronology, sequence, process, cause and effect, or transformation.
3. **Importance order** — show priority, hierarchy, rank, contribution, or progression toward a goal.

Do not combine categories unless the audience genuinely needs both relationships.

## Test the message with 4W2H

Use only the questions that serve the slide. Never force all six onto one page.

### Who?

- Extend the question to “Who is doing what?”
- Use it early in a presentation to frame people, population, disease risk, incidence, stakeholders, or affected groups.
- Prefer a large number paired with an icon.
- For two groups, separate the canvas into clear sides and pair each side with an icon or number.

### How much?

- Use direct large numerals or a restrained pictogram/icon array.
- Show scale without adding decorative units that slow counting.
- Use a chart when exact comparison matters more than an illustrative impression.

### Where?

- For geographic location, use an appropriate map such as a choropleth.
- For strategic or conceptual territory, use overlap or relationship forms such as a Venn diagram.
- Do not use a geographic map to represent a non-geographic relationship.

### When?

- Use a chronological timeline.
- Keep direction and interval logic consistent.
- Show only milestones that change the story.

### How?

- Use a cause-to-effect flow when explaining mechanism.
- Use hierarchy or taxonomy when explaining categories and subcategories.
- Use a pyramid when many elements must converge toward a goal, when upper and lower levels must be compared, or when hierarchy and sequence are both meaningful.

### Why?

- Use a pros-and-cons comparison for a focused decision.
- Use SWOT only when all four quadrants materially affect the decision.
- Conclude with the implication; do not stop at categorization.

## Chart rules

### General

- Recreate an original chart when its current specification cannot meet these rules.
- Use one hue within a chart whenever possible; vary tint or shade and reserve a different-hue accent for the key series.
- Sort chronological data chronologically. Otherwise sort ascending or descending by value according to the message.
- Prefer direct labels. If a legend is necessary, place it above the chart and arrange items left to right on one line when possible.
- Remove unnecessary borders, plot backgrounds, and gridlines.

### Donut and pie

- Prefer a donut to a pie when either form is suitable.
- Do not use either for more than five categories.
- Keep slices within one hue unless one contrasting accent is required.
- Use a bar chart when precise comparison is more important than part-to-whole impression.

### Bar

- Start the quantitative axis at zero.
- Use horizontal bars when category labels are long.
- Show values on or immediately beyond bars when readable.
- Use one hue by default and one accent bar only when it carries the message.

### Combination

- For a line-over-bar chart, use a quiet background color for bars and the main or accent color for the line.
- Keep the line visually dominant.

### Trend framing

- Avoid a plot range that makes movement misleadingly flat or dramatic.
- As a visual house target, let meaningful movement occupy roughly two-thirds of the plot height only when doing so remains statistically honest.
